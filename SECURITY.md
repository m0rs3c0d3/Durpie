# Security Policy

## Intended Use

Durpie is designed for **authorized security testing only**. 

### Acceptable Use
- Testing applications you own
- Testing with explicit written authorization
- Bug bounty programs within defined scope
- Educational purposes in lab environments
- Security research with responsible disclosure

### Prohibited Use
- Unauthorized access to any system
- Testing without explicit permission
- Disrupting services or causing damage
- Exfiltrating data without authorization
- Any illegal activity

## Reporting Security Issues

### In Durpie Itself

If you find a security vulnerability in Durpie's code:

1. Open PR
2. Include:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if any)

We will respond within 48 hours.

### Vulnerabilities Found Using Durpie

If you discover vulnerabilities while testing with Durpie:

1. **Follow responsible disclosure**
2. Report to the affected organization
3. Give them reasonable time to fix (90 days typical)
4. Do not access more data than necessary to prove the issue
5. Do not publicly disclose before the fix

## Ethical Guidelines

### Before Testing
- [ ] Get written authorization
- [ ] Define clear scope
- [ ] Agree on testing window
- [ ] Establish communication channel
- [ ] Understand local laws

### During Testing
- [ ] Stay within scope
- [ ] Don't access unnecessary data
- [ ] Avoid service disruption
- [ ] Document everything
- [ ] Stop if you cause unintended impact

### After Testing
- [ ] Report all findings
- [ ] Provide remediation guidance
- [ ] Securely delete test data
- [ ] Don't disclose without permission
